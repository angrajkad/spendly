  source venv/Scripts/activate

  venv/bin/activate is the Unix path — Windows puts the scripts in Scripts/
  instead of bin/.

❯ what does this project do?                                                    

  Read 2 files (ctrl+o to expand)

● This is a Flask-based expense tracker web app — it appears to be a
  teaching/learning project (likely for Angraj's AI learning curriculum) where
  students build the app incrementally in steps.

  Current state (starter scaffold):
  - Landing page, login, and register pages (UI only)
  - SQLite database layer (database/db.py) — not yet implemented
  - Placeholder routes for logout, profile, add/edit/delete expenses

  Planned full feature set (based on route stubs):
  - User registration & login/logout (auth)
  - User profiles
  - Add, edit, delete expenses (CRUD)

  The app is structured as a step-by-step exercise — students implement each
  feature (database setup, auth, expense management) progressively across ~9
  steps.
❯ what tech stack des this project use?                                         

● Based on the project files:

  - Backend: Python + Flask (v3.1.3)
  - Database: SQLite (via Python's built-in sqlite3)
  - Templating: Jinja2 (Flask's default)
  - Frontend: HTML/CSS + vanilla JavaScript
  - Testing: pytest + pytest-flask
  - WSGI: Werkzeug (v3.1.6)

  No frontend framework, no ORM — just plain Flask with raw SQL.

● Here's the project structure (ignoring venv/):

  expense-tracker/
  │
  ├── app.py                  # Flask app entry point — defines all routes
  │
  ├── database/
  │   ├── __init__.py         # Makes database a Python package
  │   └── db.py               # SQLite helpers (get_db, init_db, seed_db) — to
  be implemented
  │
  ├── templates/
  │   ├── base.html           # Base layout (all pages extend this)
  │   ├── landing.html        # Home/landing page
  │   ├── login.html          # Login form
  │   └── register.html       # Registration form
  │
  ├── static/
  │   ├── css/style.css       # App-wide styles
  │   └── js/main.js          # Client-side JavaScript
  │
  ├── requirements.txt        # Python dependencies
  ├── .gitignore              # Ignores venv, .db files, __pycache__, etc.
  └── venv/                   # Virtual environment (not committed)

  Key relationships:
  - app.py imports from database/db.py (once implemented) and renders templates
  from templates/
  - All HTML templates extend base.html via Jinja2 inheritance
  - static/ files are served directly by Flask and linked from the templates
